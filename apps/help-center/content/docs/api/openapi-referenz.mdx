Diese Seite spiegelt die **OpenAPI-Spezifikation** der GoodFunds-Instanz wider – dieselbe Quelle, aus
der auch die Swagger UI gespeist wird. Sie wurde aus der Spec der Gruppe „Öffentliche API"
(`platform-api`, OpenAPI 3.1) generiert und listet **60 Operationen** über 11 Bereiche.

<Callout type="info">
  Die interaktive Variante (zum Ausprobieren, mit Schemata und „Try it out") erreichst du direkt in
  der laufenden Anwendung unter **`/swagger-ui.html`**. Die rohe Spezifikation liegt unter
  **`/api/v1/api-docs`** (bzw. `/api/v1/api-docs/platform-api` für diese Gruppe).
</Callout>

<Callout type="warn">
  Generierter Inhalt – Quelle ist `platform/openapi-platform.json`. Bei Änderungen an den
  REST-Controllern die Spec neu ziehen (`platform/scripts/fetch-openapi.mjs`) und diese Seite
  aktualisieren, damit sie nicht veraltet.
</Callout>

Detaillierte Beispiele (Request-Bodies, Antworten, curl) zu den wichtigsten Flows findest du unter
[Authentifizierung](./authentifizierung) und [Endpunkte](./endpunkte). Die folgende Übersicht ist die
vollständige Referenz.

## Authentifizierung

| Methode | Pfad | Beschreibung |
| --- | --- | --- |
| `GET` | `/api/v1/auth/account-names/{userId}` | Get account names for a user (public). |
| `POST` | `/api/v1/auth/forgot-password` | Request a password reset email. Accepts email or username. Always returns success (anti-enumeration). |
| `GET` | `/api/v1/auth/load-for-edit` | Get current user's full details for the edit-profile form. |
| `POST` | `/api/v1/auth/login` | Login with username/email and password. Sets JWT cookie on success. |
| `POST` | `/api/v1/auth/logout` | Logout: revokes refresh token and clears all auth cookies. |
| `GET` | `/api/v1/auth/me` | Get current authenticated user with engagement data. |
| `POST` | `/api/v1/auth/resend-verification` | Resend email verification. Always returns success (anti-enumeration). |
| `POST` | `/api/v1/auth/reset-password` | Reset password using a token from the reset email. |
| `POST` | `/api/v1/auth/signup` | Register a new user. Creates user with UNVERIFIED status and sends verification email. |
| `POST` | `/api/v1/auth/update` | Update current user's profile. |
| `POST` | `/api/v1/auth/verify-email` | Verify email address using the token from the verification email. |

## Spenden

| Methode | Pfad | Beschreibung |
| --- | --- | --- |
| `POST` | `/api/v1/donations` | Submits a one-time donation. Returns Stripe clientSecret for confirmation. |
| `POST` | `/api/v1/donations/{intentId}/micropayment-sepa` | micropayment SEPA flow: customerCreate/bankaccountSet/sessionCreate/sessionApprove with the donor's IBAN. Intent stays PENDING until the notification confirms. |
| `POST` | `/api/v1/donations/{intentId}/micropayment-session` | micropayment card flow step 2: turn a JsBridge card token into a payment session (tokenSessionCreate). Returns the sessionId. |
| `GET` | `/api/v1/donations/{intentId}/status` | Donation status for success-page polling. Optional cryptId (micropayment card return) triggers an active transactionGetCrypt poll + reconcile. |
| `GET` | `/api/v1/donations/by-token/{token}/minimal` | Returns minimal donor + donation data (firstName, lastName, fullName, salutation, title, amount, currency, rhythm, *Cent, paymentMethodType, transactionFeeCovered) for a donation personalization token. |
